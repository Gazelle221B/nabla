# 要件: nabla(∇/ナブラ) — インタラクティブ数学学習サイト(中学〜大学教養)

> 作成者: `HUMAN`(プロダクトオーナー)。問題定義の起点。
> これがConclaveの上流。設計・実装・レビューは全てこの文書にトレースされる。
> ステータス: **v1.0 Final(凍結)**。以降の変更はIssueまたはADR経由のみ。全文の合意経緯・議論の紆余曲折は `docs/PROJECT_PLAN.md` を参照。

## 目的

独学者向けに、数学を「予想して・動かして・発見する」インタラクティブ教材を提供するWebサイトを作る。動画ではなくブラウザ内のインタラクティブ図を主軸とする。表示名ナブラ、英語名nabla、タグライン「予想して、動かして、発見する。」。

学習体験の核は、Mathigon(テキストと図解の連動)・Immersive Math(体系性と操作可能な図)・Seeing Theory(短文+即触れる図)・BetterExplained(ADEPT法)・Nicky Case/Bret Victorのexplorable explanations(問い→予想→操作→発見の黄金パターン)を統合したもの。**操作前に必ず予想させる(認知的な賭け)**ことが、単なる「触れるグラフ」との差別化点。

成果物はGitHub Pagesにデプロイされ、追加サーバーなしで永続的に公開・運用できること。

## スコープ

**Vision(最終到達目標、本要件の直接の対象外)**: 中学数学〜大学教養数学までを広くカバーする。

**MVP 1(本要件が対象とする範囲)**: 以下3記事をMafs/SVG(レンダラーTier 1)のみで作る。
- 三平方の定理(中学)
- 微分係数と接線(高校)
- 2×2行列と固有ベクトル(大学教養)

**Current scope(運用上の約束)**: 常に着手中の3〜5記事のみを具体的な約束として扱う。「いつか網羅する」と「今完成させる」を混同しない。単元の作成順序・粒度・分割は、本要件と憲法(AGENTS.md §3)の範囲内でエージェントが自律的に計画してよい。人間はマイルストーンの細分ではなく不変条件と完了の定義のみを握る。

含むもの:
- 中学数学・高校数学(数I〜III・A〜C)・大学教養数学(微積分・線形代数・確率統計)を縦につなぐカリキュラム。各単元は前提単元(prerequisite)を持つ有向グラフとして表現する。
- テキスト解説(MDX)+インタラクティブな図解コンポーネントの組み合わせによるページ群。各単元ページは操作可能な図解を最低1つ持つ。
- 単元マップ(前提関係を可視化、ノードから該当ページへ遷移)。
- 数式の高品質なレンダリング(KaTeX)。
- カリキュラムデータのスキーマ定義とビルド時検証。
- 再利用可能な図解コンポーネントの共通枠組み(ただしMVP 1では先行して汎用化しない。rule of three — 二度以上現れた構造だけを抽出する)。
- GitHub Actionsによる自動ビルド・デプロイ。

## 非スコープ

意図的に含めないもの(スコープ膨張防止の境界):

- 動画コンテンツの作成・埋め込み(図解は必ずインタラクティブなJS実装で行う)。
- サーバーサイド処理を要する機能(ログイン、ユーザー登録、進捗のサーバー保存、バックエンドAPI)。進捗保存等が必要になってもlocalStorage等クライアント完結の手段に限定し、その採用も人間承認を要する。
- 多言語対応(日本語のみ)。
- 独自ドメイン設定、アクセス解析の高度化(GA4は最小限イベントのみ)、課金、外部アカウント連携、コミュニティ/投稿機能。
- 検索機能(記事数が増えてから導入)。
- 前提知識グラフの自動生成マップ(記事20〜30本後。メタデータの記録は今から、可視化は後から)。
- **MVP 1限定の非対象**: Pixi.js / Three.js / WebGPU(レンダラーTier 2・3。詳細はDESIGN.md)、自動採点式演習問題、ライトモード。

## ユーザーストーリー

- 中学・高校生(学習者)として、教科書の文章だけでは掴みにくい概念を理解するために、図のパラメータを自分で動かして挙動を確かめられるインタラクティブな図解が欲しい。
- 大学初年次の学生(学習者)として、高校までと大学教養が分断されず、前提単元をたどって理解の穴を埋められる単元マップが欲しい。
- 独学者(学習者)として、動画を待つのではなく、自分のペースでテキストを読みながらその場で図を操作できる体験が欲しい。
- サイト運営者(=プロダクトオーナー本人)として、MDXで書き足すだけで決まったスキーマに沿って単元グラフに自動的に組み込まれ、pushするだけで公開されてほしい。

## 受け入れ条件(Acceptance Criteria)

### 基盤(プロジェクト土台)

- [ ] `main` へのpushによりGitHub Actionsが自動ビルドし、GitHub Pages上の公開URLでサイトが表示される。
- [ ] 生成物が完全な静的アセット(HTML/CSS/JS、Tier 3bのみWebGPU/WGSL関連アセット)のみで、サーバーサイド処理を一切要さない。
- [ ] カリキュラムスキーマが定義され、frontmatterのバリデーション(型・必須項目)がビルド時に機能し、不正なfrontmatterはビルドを失敗させる。
- [ ] 単元マップが各単元の前提関係を反映して描画され、ノードから該当ページへ遷移できる。
- [ ] 再利用可能な図解コンポーネントの共通枠組みが存在し、少なくとも2つ以上の単元ページで同じ枠組みが再利用されている(使い捨て実装でないことの証跡)。
- [ ] GitHub Pagesのサブパス配信下でも、全アセット・内部リンク・図解が壊れずに動作する。

### 各単元ページが満たすべき汎用条件

- [ ] テキスト解説と数式(KaTeX)が正しくレンダリングされている。
- [ ] その概念を体感できるインタラクティブ図解を最低1つ持ち、パラメータ操作(スライダー/ドラッグ等)に図がリアルタイムで追従する。
- [ ] 操作に連動して、対応する数式・数値・状態の表示が更新される、または操作結果が視覚的に確認できる。
- [ ] frontmatterが定義済みスキーマに準拠し、宣言した前提単元(prerequisite)が実在する単元IDを指している(リンク切れ・孤立ノードがない)。
- [ ] 数学的内容(定義・式・図の挙動)に誤りがないことが、実装者とは別のエージェントによって検証されている(`REVIEWER`/`QA_MEMORY`、AGENTS.md §7)。
- [ ] 対象ブラウザ(§制約参照)でレイアウト崩れなく表示・操作できる。
- [ ] ドラッグを唯一の入力にしない(スライダー/数値入力/矢印キー/リセット/現在値テキストを併設、WCAG 2.2 Level AA適用可能項目を目標)。

### MVP 1固有(Definition of Done、詳細はIMPLEMENTATION_PLAN.md)

- [ ] 学習目標が1〜2個に限定され、操作前に予想を要求し、代表的誤解を1つ以上扱い、平易な説明から形式的定義へ到達し、末尾に転用問題またはサンドボックスがある。
- [ ] 数学モデル(`lib/math`)に単体テストと不変条件テストがある(自己確認的テストは不可、REQUIREMENTS明確化欄参照)。
- [ ] axe検査でCritical/Serious 0件、Lighthouse Accessibility原則100、コンソール未処理例外0件。

## 制約・前提

- **デプロイ先はGitHub Pages固定**。静的ホスティングのため、リクエスト時実行・動的APIは利用不可。すべてビルド時生成+クライアントサイド実行で完結させる。
- **技術スタック**(詳細はDESIGN.md): Astro + MDX + React Islands + KaTeX。文章はMarkdown/MDX。図解はレンダラー3階層(Tier 1: Mafs/SVG既定、Tier 2: 高密度2DはPixi.js/WebGPU、Tier 3a: 標準3DはThree.js、Tier 3b: シェーダー品質・大量計算3DはWebGPU必須)。3DはまずThree.js(Tier 3a)で足りるか検討し、密な計算やシェーダー品質のグローが本質的に必要な場合のみWebGPU(Tier 3b)へ進む。MVP 1はTier 1のみ使用。
- **GitHub Pagesのサブパス配信**(`https://<user>.github.io/<repo>/`)を前提にbase pathを正しく扱うこと(アセットパス崩れの既知リスク)。
- **対象ブラウザ**: デスクトップ版Chrome/Edge/Firefoxの最新2メジャーバージョン、macOS Safariの現行メジャーバージョン。iOS Safari・Android Chrome現行版で本文閲覧と主要操作が破綻しないことを確認(Tier 3b=WebGPUコンポーネントのみWebGPU対応ブラウザに限定、非対応時はメッセージ表示。Tier 3a=Three.jsは他Tierと同じ対象ブラウザで動作)。
- **カリキュラム準拠**: 中学校=学習指導要領(2017告示)、高等学校=同(2018告示)。大学教養は全国統一カリキュラムが存在しないため、複数大学の初年次シラバス・標準教科書を比較した独自参照カリキュラムを定義する。次期学習指導要領の検討が進行中のため「最新準拠」ではなく参照版年をfrontmatterに明示する。
- **ライセンス**: ソースコード=GPL-3.0-or-later、MDX記事・文章・独自図版=CC BY-SA 4.0、第三者素材=各固有ライセンス。Build in public(公開リポジトリで開発)。将来非公開へ移行する可能性はあるが、既に公開・配布した版のライセンス・利用権は撤回できない。MVP期間中は外部コードPRを原則受け付けない。
- **体制・期間**: 制作者1名+AIエージェント支援。React未経験、週10時間が上限。最初の公開可能な1記事=4〜6週間。MVP 1全体=12週間目標、上限16週間。超過時は機能削減(AGENTS.md §3 C-4参照)。
- **コスト上限**: 個人プロジェクトのため厳密な期限なし。最高コストモデル(ARCHITECT)は上流設計と難所エスカレーションにのみ投入する(AGENTS.md §7)。
- **既知のリスク**: インタラクティブ図解の作り込みコストが高く、横展開時のスケールがボトルネックになりうる。図解コンポーネントの再利用パターン確立を設計時の重点とする(DESIGN.md参照)。教育効果を検証する被験者が現時点で確保できていない(§受け入れ条件からは意図的に除外し、公開ベータとして扱う)。
