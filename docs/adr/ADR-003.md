# ADR-003: ポスト MVP 1 の単元展開方針(Tier 1 波状拡張)

- **ステータス**: Accepted(制作者=HUMAN の「他の単元も全て作成」指示 2026-07-12 を Go とし、`docs/ROADMAP.md` の確定 PR マージをもって発効)
- **決定者**: ARCHITECT(Claude Code オーケストレータ)起票 → 独立レビュー(別系統)+ 制作者事後監査
- **日付**: 2026-07-12(JST)
- **関連文書**: `docs/ROADMAP.md`、`docs/REQUIREMENTS.md` §スコープ / §Current scope、`docs/DESIGN.md` §レンダリング戦略、`AGENTS.md` §3(C-4)、ADR-001

## 文脈

MVP 1(三平方の定理・微分係数と接線・2×2行列と固有ベクトルの3記事)が完了し、本番で全 DoD を実測充足した。制作者(HUMAN)は「当初の構想通り他の単元も全て作成」する方針を指示した。REQUIREMENTS の Vision は「中学〜大学教養数学を広くカバー」だが、同 §Current scope は「常に着手中の3〜5記事のみを具体的な約束として扱い、『いつか網羅する』と『今完成させる』を混同しない。単元の順序・粒度はエージェントが自律計画してよい」と定める。したがって「全単元」を**一括の実装約束**として扱うと凍結要件の運用原則に反する。

## 決定

**「全単元展開」を、`docs/ROADMAP.md` のビジョングラフへ収束していく波状(3〜5単元/波)プログラムとして実行する。** 具体的には:

1. **Tier 1 を先に埋める(C-4)**。中高範囲の大半と大学教養の一部は Mafs/SVG(Tier 1)で表現できる。Tier 2(高密度2D)/ Tier 3(3D)は、それぞれ MVP 2 / MVP 3 として**別途 Tier 導入 ADR(ADR-004 以降)+ 計測 + 人間 Go** を着手ゲートにする。本 ADR は新 Tier を導入しない。
2. **マイルストーン M4〜M7(Tier 1)** を `ROADMAP.md` の通り設定し、着手する波の3〜5単元のみを `IMPLEMENTATION_PLAN` の T タスク(終了条件+反復上限、C-7)として具体化する。
3. **各単元は MVP 1 と同一 DoD** を満たす(純粋 `lib/math`+不変条件テスト / Mafs 図解 / 予想ゲート付き対話 Island / MDX 記事 / 全ゲート緑 / a11y / 実装者≠レビュアー≠数学QA=C-5)。
4. **前提グラフ(C-2)** を各単元 frontmatter に記録し、波の順序は前提単元が先に merge されるよう直列化する。
5. **rule of three(REQUIREMENTS §62)**: 既存3実験で共通構造が確立したため、M4 で共有枠組み(予想ゲート/操作パネル/noscript)を抽出し以降で再利用する。

## 検討した代替案

| 案 | 長所 | 短所 | 採否 |
|---|---|---|---|
| **波状 Tier 1 拡張(本決定)** | 凍結要件 §Current scope に忠実。各波が独立に完成・公開でき、品質(数学正しさ・a11y)を単元単位で担保 | 全網羅までの総時間は長い(個人プロジェクトゆえ許容、REQUIREMENTS §制約) | **採用** |
| 全単元を一括生成 | 見かけ上「全部」に速く到達 | §Current scope 違反。数学的誤りの混入リスク(C-5 の別エージェント検証が形骸化)、レビュー・統合が破綻、未検証コンテンツの公開 | 不採用 |
| いきなり Tier 2/3 単元(3D/高密度)へ拡張 | 派手な表現 | C-4 違反(Tier 1 優先)。新基盤の ADR・計測なしに複雑度を上げると耐障害性・互換要件が崩れる | 不採用(MVP 2/3 で ADR を経て導入) |

## 結果

**利点**: 「全単元」という無限スコープを、要件準拠の有限な波へ分解できる。各単元が MVP 1 と同じ品質ゲートを通るため、公開物の数学的正しさ・アクセシビリティが単元単位で保証される。共有枠組み抽出で横展開コスト(既知リスク)を下げる。

**代償・注意**:
- 総完成までに多数の波を要する(1セッションで全網羅はしない)。進捗の真実は `PROJECT_STATE.md` と本ロードマップに集約する。
- Tier 2/3 が必要な単元(フーリエの大量矢印、3D 変換、複素ドメインカラーリング等)は本 ADR の対象外。MVP 2/3 の Tier 導入 ADR を待つ。
- **不可逆性**: 公開した記事のライセンス(CC BY-SA 4.0)は撤回できない(REQUIREMENTS §制約)。ゆえに数学 QA(別エージェント)を各単元のマージ前必須ゲートとする。
