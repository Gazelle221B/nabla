# ADR-002: Tier 1 図解ライブラリに Mafs を採用

- **ステータス**: Accepted(PR #4 のマージをもって確定)
- **決定者**: IMPLEMENTER(Claude Code)起票 → レビュー PASS + QA PASS + Copilot 対応を経て、新マージ規約(AGENTS.md §8, DEVELOPMENT.md §6)に基づき PR #4 のマージで確定
- **日付**: 2026-07-04(JST)
- **関連文書**: `docs/DESIGN.md` §レンダリング戦略 / §検討した代替案、`DEVELOPMENT.md` §5(AI追加依存の扱い)、`docs/IMPLEMENTATION_PLAN.md` T3-1、ADR-001

## 文脈

T3-1(`InteractiveExperiment`)は、三平方の定理の直角三角形を「予想→操作→観察」できる Tier 1 図解として実装する必要がある。`docs/DESIGN.md` は Tier 1 の既定技術を **Mafs(既定)/ SVG + D3** と確定しており(ADR-001)、直角の制約付き可動点は「Mafs の `useMovablePoint`(実装では controlled な `MovablePoint` + 制約関数)で実装可能」と明記している(§検討した代替案の `@mathigon/euclid` 行)。DEVELOPMENT.md §5 は、AI エージェントが追加する依存を無条件に受け入れず、ライセンス両立性・保守状況・数学モデルと描画層の分離を損なわないことを確認したうえで ADR に記録することを求める。本 ADR はその手続きに従い、実装中に具体的な必要性が生じた時点での Mafs 導入を記録する。

## 決定

Tier 1 図解の React コンポーネントライブラリとして **`mafs@^0.21.0`** を runtime 依存に採用する。

- 用途: 座標系(`Coordinates.Cartesian`)、制約付き可動点(`MovablePoint` + `ConstraintFunction`)、多角形・線分(`Polygon` / `Line.Segment`)、数式ラベル(`LaTeX`)。
- スタイルは `mafs/core.css` を島コンポーネント内で import し、Astro/Vite が島の CSS として束ねる。
- 数学ロジック(残差・距離)は従来どおり `src/lib/math`(純粋 TypeScript)に置き、Mafs には一切依存させない。Mafs は描画層(`src/components/scenes/mafs/`)にのみ現れ、AGENTS.md §5 の「数学モデルと描画の分離」を維持する。

## 検討した代替案

| 案 | 長所 | 短所 | 採否 |
|---|---|---|---|
| **Mafs** | React ネイティブ、可動点・座標系・KaTeX ラベルが揃い、Tier 1 の DESIGN 既定。MIT | peerDependency が React >=18 表記(React 19 では npm が ERESOLVE 警告を出すが動作する) | **採用** |
| 素の SVG を自作 | 依存ゼロ | ドラッグ・座標変換・キーボード操作・アクセシビリティを再発明することになり、DESIGN §リスク「汎用座標系・ドラッグ/ズーム/パンの自作」を避ける方針に反する | 不採用 |
| `@mathigon/euclid` | 幾何計算が揃う | 描画は別途必要で、可動点 UI を担わない。DESIGN でも「事前導入は不要な複雑化」と評価済み | 不採用(将来評価) |

## 結果

**利点**: DESIGN 既定に沿い、Tier 1 の可動点・座標系・数式ラベルを最小コードで実装できる。ライセンス(MIT)は本リポジトリのコード側(GPL-3.0-or-later)と両立する。数学と描画の分離は維持される。

**代償・注意**:
- React 19 と Mafs の peerDependency(React >=18)で `npm install` 時に ERESOLVE 警告が出る。インストール・型チェック・ビルド・テストはいずれも成功しており実害はないが、Mafs が React 19 対応を明示するまでは注視する。
- Mafs はマウントまで描画しない(SSR では図が出ない)。このため JS 無効時は図が表示されないので、`InteractiveExperiment` に `<noscript>` フォールバック文言と静的な定理記述を用意した(AGENTS.md §9「JS 無効でも本文と数式は読める」)。ビルド後の静的 HTML にフォールバックが出力されることを確認済み。
- KaTeX は Mafs も内部依存として持つ(`node_modules/mafs/node_modules/katex`)。本体の `katex` 依存(KaTeX 数式表示)とは別物で、重複は Tier 1 の範囲では問題にならない。

**不可逆性に関する注記**: Mafs の Tier 1 既定からの離脱、または Tier 境界(ADR-001)の変更は本 ADR / ADR-001 の改訂 + 人間承認を要する。
