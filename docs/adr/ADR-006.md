# ADR-006: ポスト全マイルストーンの方針転換 — M9「計測と構造化」(生産から検証・流通へ)

- ステータス: Accepted(2026-07-24、制作者 Go 受領)
- 起票: ORCHESTRATOR(Claude Code Fable 5)
- 関連: Issue #49(GPT-5.6 Sol × Kimi K3 独立協議の統合判断)、ADR-003(波状展開)、docs/memo.md(キセリョフ調査対話)

## 文脈

全マイルストーン(MVP 1〜3 + M4〜M8)+バックログ 3 件を完走し、公開 32 単元+単元マップが揃った。AI パイプラインの確立により「作る能力」が「検証する能力・届ける能力」を構造的に上回っており、未検証の設計仮説(予想→操作→発見ループ)の上に単元を量産し続けることが最大の戦略リスクになった(Issue #49 で異系統 2 モデルの独立協議が同一結論)。

教育効果検証は当初「HUMAN 主導の被験者確保」を前提としていたが、制作者判断(2026-07-24)により**被験者リクルートは行わない**(個人プロジェクトの制約)。

## 決定

次フェーズを **M9「計測と構造化」** として定義し、重心を「生産」から「検証・構造化・流通」へ移す。

### M9 のサブ波

| 波 | テーマ | 内容 | 完了条件(C-7) |
|---|---|---|---|
| **M9a** | 計測基盤 | GA4 カスタムイベント(予想入力・予想確定・操作実施・単元完了)+ 計測の合格基準を**実装前に** `docs/METRICS_PLAN.md` へ事前コミット | イベントが本番で発火・基準文書がマージ済み |
| **M9b** | 前提チェック関門(パイロット) | 代表 3 単元に前提 DAG のエッジから導出した冒頭チェック 3 問(クライアントサイド判定・不足時は前提単元へ誘導)。`lib/math` 資産を判定に転用 | 3 単元で稼働+テンプレ化判断の記録 |
| **M9c** | 演習+予想履歴 | パイロット単元に演習 5 問(即時判定・誤答パターン別フィードバック)+ localStorage 予想履歴ビュー(エクスポート可能) | パイロット単元で稼働・axe/e2e 緑 |
| **M9d** | 流通の種まき | 既定学習経路 2〜3 本(入口診断つきコース)+ 教師・伴走者向け 1 枚ガイド+印刷可能な予想ワークシート+URL パラメータでの初期状態固定 | 経路ページ公開・ガイド公開 |

各波は着手時に IMPLEMENTATION_PLAN へ T タスクとして具体化し、終了条件+反復上限を付す(C-7)。実装サイクルは確立パイプライン(IMPLEMENTER=Sonnet 隔離 worktree → 別系統独立レビュー+必要時数学 QA → CI → 統合)を踏襲する。

### P1 の適応(被験者なし検証)

- 被験者による事前/事後/遅延テストは**実施しない**。被験者確保は引き続き HUMAN ゲートとして「人間判断待ち」に残置する。
- 代替として **行動テレメトリによる破綻検出** を行う: 予想入力率・操作完了率等の実利用シグナルを GA4 で計測し、`docs/METRICS_PLAN.md` の事前登録基準(例: organic 利用者の予想入力率)と突合する。
- この計測は「効果の証明」ではなく「中核ループの致命的破綻(誰も予想を入力しない等)の検出」を目的とし、**教育効果を「検証済み」と表示しない制約(C-5)は不変**。

### やらないこと(スコープ外の明文化)

1. **単元数を KPI にした横展開**。新規単元は前提グラフの空洞埋め・経路整備に必要な場合に限定。
2. **アカウント・バックエンド・AI 家庭教師・ゲーミフィケーション**。静的ホスティングのゼロ運用コストは 1 人体制の生存戦略であり、進捗は localStorage(デバイス横断は埋めないと割り切る)。導入したくなった時点で新 ADR を必須とする。
3. 動画・教育効果の検証済み表示など既存 C 系の禁止事項(不変)。

### 製品定義の明文化

nabla は「教師を代替する完全な学習サービス」ではなく**「体系的な独学経路を備えたインタラクティブ教材基盤(システムの部品)」**とする。教師・動機づけ・認定は偽装せず、それらを持つ外部システムへの組み込みやすさ(ワークシート・埋め込みガイド・併用マップ)に責任を持つ。

## 結果

- 利点: 未検証仮説への量産投資を止め、32 単元を検証・流通の実験台として活用できる。単元マップが「見せる可視化」から「通過を要求する積み木構造」(キセリョフ知見の翻訳)へ昇格する。
- 代償: 単元数の成長が止まる(意図的)。テレメトリは効果の証明にならない(破綻検出のみ)。localStorage 進捗はデバイスを跨げない。
- C-4 との関係: 本 ADR が M9 のスコープ境界を定義する。M9 の DoD 充足まで Tier 拡張・新単元波・上記「やらないこと」に着手しない。
- GA4 は MVP 1 の削減リスト(C-4)に含まれていたが、削減は MVP 1 期間の措置であり、M9a での復活は本 ADR を根拠とする。
